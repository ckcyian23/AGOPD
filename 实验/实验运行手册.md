## 1. 当前实验目标

当前目标是完成 LR-TIP 的完整 offline validation，并为后续 OPD/TIP 训练集成准备可信证据。

关键修正：

- TIP 不是 KL-only。
- TIP 是 student entropy 与 teacher-student divergence 的二维 taxonomy。
- 完整 TIP selection baseline 使用 Soft-OR：

```text
TIP Soft-OR = 1 - (1 - H_norm) * (1 - D_norm)
```

- 当前 LR-TIP full score 使用：

```text
LR-TIP Soft-OR = 1 - (1 - H_norm) * (1 - D_norm) * (1 - R_norm)
```

旧结果仍可作为 `divergence-only + relation` 的 sanity check，但不能再称为完整 TIP baseline 对比。

## 10. 结果解析重点

每个 report 需要记录：

- `mean_do_dr_corr`
- `mean_entropy_do_corr`
- `mean_entropy_dr_corr`
- `entropy_only_*`
- `div_only_*`
- `tip_*`
- `lr_tip_*`
- `lr_tip_full_*`
- `entropy_tip_overlap`
- `div_tip_overlap`
- `tip_lr_tip_overlap`
- `tip_lr_tip_full_overlap`

最关键的判断：

1. `D_R` 是否与 entropy / divergence 低相关。
2. `LR-TIP full` 是否在 `D_R` 上明显高于 `TIP Soft-OR`。
3. `LR-TIP full` 的 `D_O` 是否仍高于 random。
4. `LR-TIP full` 与 `TIP Soft-OR` overlap 是否不是 1。
5. window/layer/KL direction 改变后结论是否稳定。

## 12. 下一步执行顺序

1. synthetic 1024 prompt 只保留为工程吞吐参考，不再作为主论文证据。
2. 主线切换到真实 benchmark prompt：
   - `data/real_prompts_512.jsonl`
   - GSM8K 256 prompts
   - MATH-500 256 prompts
3. 第一优先级验证 `D_R` 是否预测 gradient-norm teacher improvement：
   - 对 token `t` 的 teacher/student logits 计算蒸馏 KL loss。
   - 对 student 参数执行一次 `backward()`，但不更新参数。
   - 记录 `sum(p.grad.float().square().sum() for p in student.parameters())`。
   - 每个 token 后执行 `student.zero_grad(set_to_none=True)`。
   - 依据是一阶泰勒展开：固定学习率下，单步 loss 下降潜力与梯度范数平方成正比。
4. 当前正在运行：
   - `outputs/lr_tip_grad_improvement_real32_w16`
   - 启动脚本：`scripts/launch_lr_tip_grad_improvement_real32.sh`
   - 配置：真实数据 32 prompts，每条采样 16 target tokens，总计约 512 次 backward。
5. 如果 `D_R` 与 gradient improvement 有正相关，并且 LR-TIP selectors 选中的 improvement 高于 TIP/random，再进入融合策略验证：
   - `S = SoftOR(H_norm, D_norm, gamma * R_norm)`
   - `S = TIP_SoftOR + lambda * R_norm`
   - `S = TIP_SoftOR + lambda * R_norm * (1 - TIP_SoftOR)`
6. 对应命令行参数：
   - `--lr-tip-full-mode soft_or --relation-gamma <value>`
   - `--lr-tip-full-mode add --relation-lambda <value>`
   - `--lr-tip-full-mode gated --relation-lambda <value>`
7. 当前 gradient improvement 验证已经显示：
   - `D_R` 与 gradient-improvement 有正相关。
   - additive LR-TIP 在低预算 top selection 中优于 TIP。
   - parameter-free Soft-OR 提升很小，不应作为唯一方法形态。
8. 下一步做小规模蒸馏提升验证：
   - 固定 teacher/student/model/data
   - 对比 random / TIP / LR-TIP selected tokens
   - 观察 held-out teacher KL 或 NLL 是否改善
   - 如果 LR-TIP proxy 强但 held-out 不提升，必须回头修改 selection 或训练目标
9. 当前 uplift pilot：
   - 脚本：`agopd/experiments/distillation_uplift_eval.py`
   - 输出：`outputs/lr_tip_distill_uplift_real16_budget005`
   - 对照：random / divergence / TIP / LR-TIP additive
   - 指标：`eval_kl_after - eval_kl_before`
10. 只有真实数据 + gradient-improvement correlation + 小规模蒸馏都成立后，再考虑更大数据集和更大模型。
