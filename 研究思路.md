## LR-TIP 当前最终思路总结（论文版）

### 方法名称

**LR-TIP: Local Relational Token Importance for Efficient On-Policy Distillation**

核心思想：

> 现有 token selection 方法主要从 output space 衡量 teacher 和 student 的差异，而 LR-TIP 引入第二个观察空间：local relational representation space，通过衡量 teacher/student 局部表示关系的一致性，发现仅靠输出差异无法捕获的重要 token。

---

# 1. Motivation

现有 TIP 不是单纯的 KL token selector，而是二维 token taxonomy：

[
D_H(t)=H(P_S^t)
]

和：

[
D_O(t)=D_{KL}(P_T^t||P_S^t)
]

其中 entropy 轴表示：

> student 在 token t 是否不确定。

divergence 轴表示：

> teacher 和 student 在 token t 的预测分布是否一致。

TIP 的核心结论是：

* high-entropy tokens 有学习信号；
* low-entropy + high-divergence tokens 是 entropy-only 会漏掉的 overconfident error；
* Soft-OR 可以融合 entropy 和 divergence：

[
S_{TIP}(t)=\hat H_t+\hat D_t-\hat H_t\hat D_t
]

问题：

两个模型可能：

* output probability 接近
* 但内部 contextual representation organization 不同

因此：

output agreement 不一定代表 representation alignment。

所以需要一个互补 signal：

**local relational representation consistency**

---

# 2. 核心设计

## 2.1 不比较 hidden vector

不使用：

[
||h_t^T-h_t^S||
]

原因：

### (1) Architecture gap

teacher/student：

* hidden dimension 不同
* layer 数不同

无法直接比较。

### (2) Representation space mismatch

hidden space 存在 rotation / transformation。

absolute distance 不稳定。

因此：

比较：

> token 与上下文之间的关系结构。

---

# 3. Local Relational Representation

对于当前 token：

[
t
]

取局部历史窗口：

[
C_t=
{t-m_t,...,t-1}
]

其中：

[
m_t=min(w,t-1)
]

例如：

[
w=16
]

只考虑最近 16 个 token。

---

## Teacher relational profile

计算：

[
r^T_{tj}
========

cos(h_t^T,h_j^T)
]

得到：

[
R_T(t)
======

[
r^T_{t,t-m_t},
...
,r^T_{t,t-1}
]
]

表示：

teacher 如何组织 token t 与历史 context 的关系。

---

## Student relational profile

同理：

[
r^S_{tj}
========

cos(h_t^S,h_j^S)
]

得到：

[
R_S(t)
]

表示：

student 的 local contextual organization。

---

# 4. Local Relational Discrepancy

定义：

[
D_R(t)
======

\frac1{m_t}
\sum_{j=t-m_t}^{t-1}
Huber(
r^T_{tj}-r^S_{tj}
)
]

含义：

衡量：

> teacher 和 student 对 token t 周围上下文关系建模方式的偏离程度。

特点：

* 不需要 hidden dimension 对齐
* 不需要 layer matching
* 不需要 distribution normalization
* 不需要 ranking/sorting

---

# 5. Cold Start

当：

[
t=1
]

没有历史 token：

[
D_R(t)=0
]

只使用：

[
D_O(t)
]

当：

[
t<w
]

自动：

[
m_t=t-1
]

无需 padding。

---

# 6. Dual-view Token Importance

最终 importance：

[
I_t=
\alpha D_O(t)
+
\beta D_R(t)
]

注意：

这是 LR-TIP 的 divergence-relation 版本，用于验证 `D_R` 是否提供互补信息。

如果要和完整 TIP 公平对比，应该使用 entropy-aware 版本：

[
S_{LR-TIP}(t)=
1-(1-\hat H_t)(1-\hat D_t)(1-\hat R_t)
]

即在 TIP 的 entropy + divergence Soft-OR 上加入 local relation discrepancy。

其中：

### Output disagreement

[
D_O(t)
]

表示：

> student 输出预测哪里偏离 teacher。

### Relational disagreement

[
D_R(t)
]

表示：

> student 内部局部 contextual representation 哪里偏离 teacher。

二者互补。

---

# 7. Token Selection

根据：

[
I_t
]

排序：

选择：

[
Top-k(I_t)
]

用于：

**On-Policy Distillation**

只对重要 token 使用 teacher supervision。

# 8. 计算复杂度

窗口大小：

[
w
]

每个 token：

需要：

[
w
]

次 cosine。

因此：

Time:

[
O(Twd)
]

Memory:

缓存最近：

[
w
]

hidden states。

避免：

global relational modeling:

[
O(T^2d)
]

# 9. 方法贡献

## Contribution 1

提出新的 token importance signal：

从：

[
output\ space
]

扩展到：

[
local\ relational\ representation\ space
]

---

## Contribution 2

提出：

**Local Relational Discrepancy**

一种轻量级 representation alignment metric：

避免：

* hidden vector comparison
* global relation computation
* quadratic complexity

---

## Contribution 3

提出：

**Dual-view Token Selection**

融合：

[
Output\ Disagreement
+
Local\ Relational\ Disagreement
]

相比传统 TIP：

能够发现：

output-level disagreement 无法捕获的重要 token。

---

# 10. 论文核心一句话

> LR-TIP identifies important tokens by measuring both output-level disagreement and local relational representation inconsistency between teacher and student, providing a complementary view for efficient token selection in on-policy distillation.

---

# 11. 当前需要证明的三个关键点

## Experiment 1：Signal Complementarity

证明：

[
D_R
]

不是：

[
D_O
]

的重复。

验证：

* correlation
* overlap of selected tokens

目标：

证明：

两者捕获不同信息。

---

## Experiment 2：Fixed Budget Advantage

固定 distillation budget：

例如：

10%、20% tokens。

比较：

* Random
* TIP
* LR-TIP

证明：

同样 teacher cost 下：

LR-TIP 更有效。

---

## Experiment 3：Robustness

验证：

不同：

* teacher/student size gap
* model family
* window size

例如：

70B → 7B

证明：

方法不是依赖同结构模型。

---

## 当前版本定位

不是：

“新的 attention score”

而是：

> **A second-view token importance estimator that complements output disagreement with local relational representation consistency.**

这个定位目前是最清晰、最容易 defend 的版本。
